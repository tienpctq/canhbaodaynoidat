QUẢN LÝ THIẾT BỊ

Dữ liệu được đồng bộ realtime từ Firebase path /Device. Mỗi thiết bị dùng key như ID1_001 và các trường ESP32: LINE1, LINE2, LINE3, LINEPW, ONLINE, WARNING, ENABLEBUZZ, RST, RESPONSE, SDT, SMS, SEND_INTERVAL, VITRI.

CẤU HÌNH HỆ THỐNG

Realtime Database hiện yêu cầu người dùng đã xác thực. Web tự đăng nhập Firebase Anonymous Auth trước khi đọc/ghi dữ liệu.

{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}

Cần bật Firebase Console → Authentication → Sign-in method → Anonymous. Nếu ESP32 đọc/ghi trực tiếp, firmware cũng cần cơ chế xác thực tương ứng.

NGƯỜI DÙNG

adminQuản trị hệ thống web
ESP32Thiết bị gửi trạng thái qua Firebase

PHÂN QUYỀN

Hiện web dùng Anonymous Auth để thỏa rule auth != null. Khi vận hành chính thức nên chuyển sang Email/Password hoặc custom token để phân quyền admin và thiết bị.

SAO LƯU DỮ LIỆU

Xuất toàn bộ dữ liệu đang đọc được từ Firebase thành file JSON.

GIỚI THIỆU

Hệ thống giám sát dây nối đất dùng ESP32, Firebase Realtime Database và web dashboard Netlify.
